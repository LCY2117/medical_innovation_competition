# LifeReflexArc 项目代码描述

## 项目概述

LifeReflexArc 是一个应急协作平台，用于在紧急情况下（如心脏骤停）协调多方救援力量。系统包含 Android App 端、云端 Python 后端服务和 React Web 前端三部分。

---

## 一、Android App 端（Kotlin + Jetpack Compose）

### 1.1 数据层 (`data/`)

| 文件 | 描述 |
|------|------|
| `ApiService.kt` | Retrofit 接口定义，负责与后端 REST API 通信（注册、登录、事件管理） |
| `WsClient.kt` | WebSocket 客户端，实现与服务器的实时双向通信，接收事件更新 |
| `AuthRepository.kt` | 认证仓库，管理用户登录/注册的业务逻辑 |
| `IncidentRepository.kt` | 事件仓库，管理应急事件的数据操作 |
| `UserSession.kt` | 用户会话管理，维护认证令牌和用户信息 |
| `AuthModels.kt` | 认证相关的数据模型（登录请求、令牌等） |
| `IncidentModels.kt` | 事件相关的数据模型（事件、角色、状态等） |

### 1.2 业务逻辑层 (`viewmodel/`)

| 文件 | 描述 |
|------|------|
| `SessionViewModel.kt` | 管理用户会话、认证状态、用户信息 |
| `IncidentViewModel.kt` | 管理当前事件状态、角色分配、实时更新 |

### 1.3 UI 层 (`ui/`)

| 文件 | 描述 |
|------|------|
| `AppRoot.kt` | 主应用容器，根据认证状态路由显示不同屏幕 |
| `ScreenRouting.kt` | 屏幕导航路由管理 |
| `PhoneAppRoot.kt` | 手机界面根组件 |
| `PhoneFrame.kt` | 手机界面框架 |

### 1.4 屏幕组件 (`ui/screens/`)

| 文件 | 描述 |
|------|------|
| `LoginScreen.kt` | 登录界面 |
| `EmergencyAlertScreen.kt` | 紧急警报屏幕 |

## 二、云端服务（Python + FastAPI）

### 2.1 核心入口

| 文件 | 描述 |
|------|------|
| `app/main.py` | FastAPI 应用工厂，初始化服务、中间件、路由 |
| `app/cli.py` | 命令行接口 |

### 2.2 API 层 (`app/api/`)

| 文件 | 描述 |
|------|------|
| `app/api/rest.py` | REST 路由，处理事件创建、用户认证、客户端注册、事件加入、调度等 HTTP 请求 |
| `app/api/ws.py` | WebSocket 路由，实现实时双向通信和状态广播 |

### 2.3 业务服务层 (`app/services/`)

| 文件 | 描述 |
|------|------|
| `app/services/auth.py` | 认证服务，用户注册、登录、令牌管理 |
| `app/services/incidents.py` | **核心业务服务**：管理事件生命周期、角色分配、客户端注册、实时状态广播、WebSocket 连接管理 |
| `app/services/dispatch_ai.py` | AI 调度规划服务，调用 SiliconFlow 大语言模型进行智能调度 |

### 2.4 数据模型 (`app/models/`)

| 文件 | 描述 |
|------|------|
| `app/models/schemas.py` | Pydantic 数据模型定义，包括认证、事件、客户端、调度等请求/响应结构 |

### 2.5 数据持久化 (`app/storage/`)

| 文件 | 描述 |
|------|------|
| `app/storage/sqlite_store.py` | SQLite 事件存储实现，负责事件状态持久化 |
| `app/storage/sqlite_auth_store.py` | SQLite 认证存储实现，管理用户凭证 |

### 2.6 核心配置 (`app/core/`)

| 文件 | 描述 |
|------|------|
| `app/core/config.py` | 应用配置管理，环境变量读取 |
| `app/core/frontend.py` | 前端静态文件挂载 |

## 五、核心功能特性

- **用户认证**：注册、登录、令牌管理
- **事件管理**：创建紧急事件、状态追踪、历史归档
- **角色分配**：患者（Prime）、指导者（Guide）、跑步者（Runner）等动态角色
- **实时协作**：WebSocket 实时多用户状态同步
- **CPR 指导**：节拍器、阶段提示、计时功能
- **AI 调度**：基于大语言模型的智能资源分配和协调建议
- **健康监测**：实时客户端状态、在线/离线追踪
